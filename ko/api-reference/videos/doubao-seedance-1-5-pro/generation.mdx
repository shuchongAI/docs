## 요청 매개변수

<ParamField body="model" type="string" required>
  영상 생성 모델 이름

  지원되는 모델:
  - `doubao-seedance-1-5-pro` - 1.5 Pro 버전, 오디오 생성 및 다중 참조 이미지 지원
</ParamField>

<ParamField body="prompt" type="string" required>
  영상 콘텐츠 설명

  더 나은 생성 결과를 위해 장면, 동작, 스타일 등을 자세히 설명하세요

  예: `"해변의 일몰, 바다 위의 황금빛 햇살, 모래사장을 부드럽게 치는 파도"`
</ParamField>

<ParamField body="duration" type="integer" default="5">
  영상 길이(초)

  지원 범위: `4` ~ `12` 초

  기본값: `5`
</ParamField>

<ParamField body="aspect_ratio" type="string" default="16:9">
  영상 화면 비율

  옵션:
  - `16:9` - 가로 화면
  - `9:16` - 세로 화면
  - `1:1` - 정사각형
  - `4:3` - 전통적인 비율
  - `3:4` - 세로 전통적인 비율
  - `21:9` - 울트라 와이드
  - `adaptive` - 어댑티브 (이미지 업로드 시에만 사용 가능)

  기본값: `16:9`
</ParamField>

<ParamField body="resolution" type="string" default="720p">
  영상 해상도

  옵션:
  - `480p` - 표준 화질
  - `720p` - HD 화질

  기본값: `720p`

  <Warning>
  **1.5 Pro 해상도 제한**: 1.5 Pro는 1080p 해상도를 지원하지 않습니다. 최대 720p까지 지원됩니다
  </Warning>
</ParamField>

<ParamField body="seed" type="integer">
  생성된 콘텐츠의 무작위성을 제어하기 위한 시드 정수

  값 범위: `-1`에서 `2^32-1` 사이의 정수

  <Note>
  - 동일한 요청에서 모델이 다른 시드 값을 받으면(예: 시드를 지정하지 않거나 시드를 -1로 설정하면 임의의 숫자가 사용됨) 다른 결과가 생성됩니다
  - 동일한 요청에서 모델이 동일한 시드 값을 받으면 유사한 결과가 생성되지만 완전히 동일하다는 것은 보장되지 않습니다
  </Note>
</ParamField>

<ParamField body="audio" type="boolean" default="false">
  오디오 생성 여부

  `true`로 설정하면 영상에 AI 생성 오디오가 포함됩니다

  기본값: `false`

  <Note>
  오디오 생성은 1.5 Pro 전용 기능입니다. 1.0 버전은 이 매개변수를 지원하지 않습니다
  </Note>
</ParamField>

<ParamField body="camerafixed" type="boolean" default="false">
  카메라 고정 여부

  `true`로 설정하면 카메라 위치가 고정됩니다

  기본값: `false`
</ParamField>

## 해상도 및 화면 비율 조합

| 해상도 | 지원되는 화면 비율 | 참고 |
|--------|-------------------|------|
| 480p | 16:9, 4:3, 1:1, 3:4, 9:16, 21:9 | 모두 지원 |
| 720p | 16:9, 4:3, 1:1, 3:4, 9:16, 21:9 | 모두 지원 |

<ParamField body="image_urls" type="array">
  이미지에서 영상으로 변환을 위한 이미지 URL 배열

  자동 역할 할당 규칙:
  - 1장 = 첫 프레임
  - 2장 = 첫 프레임 + 마지막 프레임
  - 3장 이상 = 첫 프레임 + 마지막 프레임 + 참조 이미지

  예: `["https://example.com/first.png", "https://example.com/last.png"]`

  <Warning>
  - `image_urls`와 `image_with_roles`는 동시에 사용할 수 없습니다
  </Warning>
</ParamField>

<ParamField body="image_with_roles" type="array">
  더 정밀한 제어를 위한 역할 지정 이미지 배열

  <Expandable title="필드 설명">
    <ParamField body="url" type="string" required>
      이미지 URL 주소
    </ParamField>

    <ParamField body="role" type="string" required>
      이미지 역할

      옵션:
      - `first_frame` - 첫 프레임 이미지, 영상 시작 프레임으로 사용(1장만 지원)
      - `last_frame` - 마지막 프레임 이미지, 영상 종료 프레임으로 사용(1장만 지원)
      - `reference_image` - 참조 이미지, 영상 스타일 가이드용(1-4장 지원)
    </ParamField>
  </Expandable>

  예:
  ```json
  [
    {"url": "https://example.com/start.png", "role": "first_frame"},
    {"url": "https://example.com/end.png", "role": "last_frame"},
    {"url": "https://example.com/ref1.png", "role": "reference_image"},
    {"url": "https://example.com/ref2.png", "role": "reference_image"}
  ]
  ```

  <Warning>
  - `image_urls`와 `image_with_roles`는 동시에 사용할 수 없습니다
  - 첫 프레임과 마지막 프레임은 각각 1장만 지원
  - 참조 이미지는 1-4장 지원, `reference_image` 역할 사용(참고: 1.0 버전은 `reference` 사용, 1.5 Pro는 `reference_image` 사용)
  </Warning>
</ParamField>

## 1.0 버전과의 차이점

| 기능 | 1.0 fast/quality | 1.5 Pro |
|------|------------------|---------|
| 기본 해상도 | 1080p | **720p** |
| 지원 해상도 | 480p/720p/1080p | **480p/720p** |
| 길이 범위 | 2-12초 | **4-12초** |
| 오디오 생성 | 미지원 | **지원** |
| 참조 이미지 역할 | `reference` (1장) | **`reference_image` (1-4장)** |
