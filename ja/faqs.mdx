## 接続と使用方法

### APIMartサービスに接続できない場合は？

APIMartサービスに接続できない場合は、以下を確認してください：

- API Keyが正しく設定されているか確認
- ネットワーク接続が正常か確認
- APIエンドポイントアドレスが正しいか確認：`https://api.apimart.ai`
- ファイアウォールやプロキシ設定が接続をブロックしていないか確認

### モデル選択後に応答がないまたはエラーが発生する？

これは以下の理由が考えられます：

- API Keyが正しく設定されていないか期限切れ
- 選択したモデルが一時的に利用できない
- リクエストフォーマットが正しくない
- アカウント残高が不足

[APIMartダッシュボード](https://apimart.ai)でアカウントステータスとAPIキーを確認してください。

### 対話エラーや中断が発生した場合は？

対話エラーが発生した場合：

- エラーメッセージの具体的な情報を確認
- リクエストパラメータがAPI仕様に準拠しているか確認
- APIドキュメントで正しいリクエストフォーマットを確認
- 問題が続く場合は、テクニカルサポートにお問い合わせください

## アカウント管理

### API使用状況と費用を確認するには？

使用状況と費用は以下の方法で確認できます：

1. [APIMartダッシュボード](https://apimart.ai)にログイン
2. コンソールで使用統計を確認
3. 詳細な呼び出し記録と料金明細を確認

### テクニカルサポートを受けるには？

テクニカルサポートが必要な場合は、以下の方法でお問い合わせください：

- [Discordコミュニティ](https://discord.gg/V8zqssyZ5c)に参加
- [Twitter @APIMart_](https://x.com/APIMart_)をフォロー
- WeChatカスタマーサービスを追加（ダッシュボードのQRコードをスキャン）
- テクニカルサポートチームにメールを送信

## セキュリティと設定

### API Keyの管理と保護方法は？

API Keyのセキュリティを保護する：

- 公開コードリポジトリにAPI Keyをコミットしない
- 環境変数を使用してAPI Keyを保存
- 定期的にAPI Keyをローテーション
- API Keyが漏洩した場合は、ダッシュボードですぐに再生成

## 機能と特性

### どのようなAIモデルがサポートされていますか？

APIMartは様々な主流AIモデルをサポートしています：

- **テキストモデル**：GPT-5、Claude、Geminiなど
- **画像モデル**：GPT-4o-image、Gemini-2.5-Flash-Image-previewなど
- **動画モデル**：Sora2、VEO3など
- **オーディオモデル**：Whisper-1、TTSなど

サポートされている具体的なモデルについては、[APIドキュメント](/ja/index)をご確認ください。

### OpenAI APIとの互換性は？

APIMartはOpenAI APIフォーマットと完全に互換性があります：

- APIエンドポイントを直接置き換え可能
- 同じリクエストフォーマットを使用
- 既存のコードを変更する必要なし
- すべての主要なOpenAI API機能をサポート

### API応答が公式ウェブ版（ChatGPT、Claudeなど）と異なるのはなぜですか？

同じ基盤モデルを使用していますが、公式ウェブ版とAPI呼び出しには違いがあります：

**公式ウェブ版**には多くのエンジニアリング最適化と強化機能が含まれています：
- リアルタイム情報取得のための内蔵インターネット検索
- コンテキストを自動的にリンクする会話メモリ
- 内蔵計算機とコード実行環境
- 事前設定されたシステムプロンプト
- 最適化されたユーザーインターフェース

**API呼び出し**は生のモデル機能を提供します：
- コア推論と生成能力のみ
- 開発者が会話履歴を手動で管理する必要がある
- システムプロンプトとコンテキストの手動設定が必要
- ツール呼び出し（Function Calling）の自己統合が必要

<Tip>
**開発のヒント**：APIを使用する際は、システムプロンプト（system message）を積極的に設定し、会話履歴を維持し、必要に応じてツール呼び出し機能を統合することで、ウェブ版に近い体験を実現できます。
</Tip>

## コスト最適化

### API呼び出しコストを最適化するには？

コスト最適化のための提案：

- ニーズに適したモデルを選択（モデルによって価格が異なります）
- ストリーミングレスポンスを使用して待機時間を短縮
- 合理的なトークン制限を設定
- 一般的なレスポンスをキャッシュ
- 使用状況を監視して予期しない消費を回避
